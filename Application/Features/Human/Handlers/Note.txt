* Cái folder này bản chất được tạo ra là để:
	- Phòng trường hợp quá nhiều commands và queries => đưa handlers lại 1 folder cho gọn
	- Tuỳ kiến trúc ban đầu => có thể biến thể tuỳ ý
	- 

* Giải ngố CQRS Pattern:
	→ So sánh trực quan với Repository + Service pattern mà hay code
		- Commands & Queries ~ Service Interfaces
		- Handlers ~ Services Implementations
		("~": tạm hiểu nghĩa là gần như tương đương chứ ko chính xác là ngang nhau đâu)
	=> tạm hiểu được là CQRS ~ Services được chia nhỏ

		(khác so với public class bên java hay cpp)
		→ Có demo cho DTOs vs Validators
